Tailwind CSS

Tailwind CSS är ett populärt CSS-ramverk som fokuserar på en så kallad utility-first-struktur, där man bygger designen med hjälp av små, färdiga klasser men utan fördefinierade komponenter. Det kan installeras via CLI-verktyget, som ett PostCSS-plugin för integration med verktyg som Parcel och Webpack, eller importeras via CDN för snabb testning i webbläsaren (framför allt för utvecklingssyften).

Tailwind kan hämtas från: https://tailwindcss.com/

Tankar om användningen

Jag tyckte att det var lite krångligt att installera Tailwind även fast att instruktionerna vid första anblick såg enkla ut. Jag använder Parcel i det hjär projektet och följde installationsanvisningarna som fanns där men hade ändå problem ett tag innan jag fick det att fungera. Jag vet egentligen inte vad det var som krånglade men jag fick sitta och prova lite olika varianter tills det fungerade.

Tailwind CSS är mycket friare i sin användning i jämförelse med Bootstrap men det blir rörigt och lite överväldigande att se alla olika klasser i html-koden. Jag tycker att det är svårt att snabbt få en överblick vad man behöver ändra när man sitter med designen, utan att råka skriva samma klass två gånger eller för att säkerställa att man inkluderat klasser för olika skärmstorlekar exempelvis. Det var ganska lätt att hitta i dokumentationen efter själva installationen, men lite tråkigt att det finns många färdiga komponenter som är bakom en betalvägg (tailwindui.com) istället för fritt tillgängliga som i Bootstrap. Men även i det här ramverket är det ju skönt att responsiviteten inkluderas.

Jag hittade extensionen Tailwind CSS IntelliSense i VSC som jag tyckte underlättade mycket i form av att man får upp auto-complete förslag och att man kan hålla över klasserna och se vad de gör. Jag är lite kluven i vad jag tycker om Tailwind CSS, å ena sidan tycker jag om att det är så fritt och att man kan sätta ihop det som man vill mycket enklare än i Bootstrap. Å andra sidan tycker jag att koden blir jobbigare att arbeta med, framför allt när man suttit några timmar vid datorn, för att se och hålla isär vad de olika klasserna gör. Men jag anar att det är som med det mesta; att om man använder det mer så lär man sig vad saker heter och lättare kan scanna av koden under utveckling.
